## 2. v1.3 新发现的结构性不足

### 高严重性 × 大范围

1. **B1→B2 闭环缺失。** §4.3 要求 `V(x)` 跨 `x` 覆盖、低 `x` 稀疏披露，并检验披露策略不变性；§4.6 却只写“分箱下限达标”。没有每箱有效样本/设计效应、`V` 不变性、模型家族一致性、任务污染审计和失败后延长/降级规则。**[E]**
2. **B2→B3 闭环缺失。** 没有评估集满足 MDE、主效应不确定性、跨模型/任务流稳健性和阴性结果的出口表；“负结果必报”是写作承诺，不是流程出口。参考范式将外推失败写入结果（`reference-paper-anatomy.md` §5.2–§5.3），本路线图未定义失败后论文形态。**[E]**
3. **关键承诺没有交付人。** A 的 H4-ii 悬念句是 B 的咬合锚点（§2、§6.7），但 §8 只有人工交叉核对；“周级自动对账”（§4.8）没有产物、告警人、SLA。B3 也没有 trace 封存、权限、复算环境和投稿失败后的结束态。**[E]**

### 高严重性 × 中范围

1. **闸门只测管线健康，不测可识别性。** B0.5 的完整率/解析率/时延/成本不能证明信息集操纵有效、聚簇有效 `n` 足够或任务流未泄漏。**[E]**
2. **A/B 是双向脆弱依赖。** §6.7 只写 A 被删句会损害 B；实际上 B 的信息集定义可能反向收窄 A 的 H4 措辞，二者还共享作者、来源和审核资源。§5 的“不依赖 A 先发表”只解除顺序依赖，不解除语义、版本和资源依赖。需独立 OSF 与共享契约。**[E]**
3. **外部依赖无替代矩阵。** API 价格/配额、模型下线、OSF 政策、Polymarket/结算 schema、开源权重环境都未规定替代和切换是否改变确认性问题。**[H→E]**
4. **数据生命周期缺失。** §4.2 的“trace 当场全量记录”未规定存储、访问、备份、脱敏、发布 manifest、保留年限和失效声明；“可复现”因而没有长期对象。**[E]**

### 中严重性 × 大范围：监控与可证伪性

- §4.8 三类失败未覆盖 `actual_agent_count`、超时/重试、同日聚簇、任务重复/开发集泄漏、检索污染、结算来源变更、版本不变但行为漂移。任务流污染和群体合谋目前是**[H]**，不应写成已发生事实，但应记录可观测信号。
- “`V(x)` 充分统计量”“信息集是区分跟风与独立同答的唯一干净手段”（§4.1–§4.2）超出可证伪范围：前者缺不变性检验，后者只能识别暴露市场信息的效应，不能证明内部信息独立。herding 指标应限定为行为趋同/信息集效应，而非真实“市场回声”。**[E]**

## 3. v1.3 过度设计（成本 × 收益）

1. **探索性旋钮仍有隐藏 factorial（高成本/低收益）。** 确认性已减为两个，但人设、四类聚合、完整/加噪/延迟信息、300 探针、辩论 3–5 轮、≥2 家族和三类任务路径叠加后仍会爆炸；“探索性”不等于零成本。保留默认人设、一个默认聚合和一个探索臂，其余 future work。**[E]**
2. **B0 四份“一页纸”有仪式成本（中成本/中收益）。** schema+完整性测试是硬依赖；SOP、调度、结算裁定可合为一份运行/结算决策书。保留四类内容，改为两份可执行合同，不追求文档数量。**[E]**
3. **成本四层防护边界重复（中成本/中收益）。** 估算、pilot 偏差、运行阈值、降级规则分别对应预算/测量/停机/范围控制，并非全冗余；问题是没有统一 `warning/hard_cap/owner`，容易成为机械式严肃。合并为一份预算合同。**[E]**
4. **每日 git hash 链可能是低增量协议（低成本/低收益）。** 单团队已取消 commit-reveal（§4.2）；若采用不可变存储+发布 manifest，自动 hash 足够，人工日更没有额外信号。**[E]**
5. **统计诊断套件有 cargo-cult 风险（中成本/不确定收益）。** mixed-effects、bootstrap、CV、残差/VIF/正则化全套来自参照范式；应按最终模型复杂度触发，不应为“像 Scaling Agent Systems”而默认执行。**[E]**
6. **补充人工归因的 κ≥0.80（中成本/低边际收益）。** §4.4 已规定自动指标承重；若人工编码只作 illustrative，应报告抽样 κ 而不设硬阻塞，κ 硬门槛只适用于承重归因。旧 SE review 的 B4–B6 在 v1.3 记录为未裁定，不能默认为必需。**[E]**

## 4. 改进建议

### P0（阻塞性）

1. **补 B1/B2 出口闸门**（定位：§4.3–§4.6；成本：文本 0.5–1 天，执行 2–5 天）：分别冻结有效样本/设计效应、`V` 不变性、污染审计、MDE、模型家族稳健性和阴性结果路径。
2. **B0 默认生成独立 OSF+共享契约**（§2、§6.7、§8；1–2 天）：A 引用降为动机增强，不成为 B 的识别依赖；锁定术语、版本、冻结日、删改规则。
3. **建立 trace/data lifecycle contract**（§4.2、§4.8；2–4 天）：强制 `schema_version/run_id/actual_agent_count/retry_count/source_snapshot_id/settlement_version`，定义原始/分析/公开三层、权限、备份、保留期和恢复演练。
4. **把“充分统计量/最优策略”改为条件假设**（§4.2–§4.3；1–3 天）：若不变性或独立评估失败，只报告响应曲线，不称最优。

### P1（返工风险）

5. **把 §4.8 改成可观测性矩阵**（故障—信号—阈值—动作—owner—恢复；2–4 天），补齐有效 `n`、超时、泄漏、结算 schema、行为漂移和相关性漂移。
6. **做外部依赖替代矩阵并把模型家族/任务流列为区组**（1–2 天设计）；不可替代项预写降级。
7. **将 B0 四件交付物合为两份可执行合同**（约 1 天），每份附 demo fixture/失败样例。

### P2（优雅性）

8. **探索臂设硬预算和停止规则**（0.5 天）：300、辩论、加噪/延迟三者择一。
9. **统计诊断按模型触发**（0.5 天），并统一页眉 v1.3 与 §9 已出现的 v1.4 版本记录（0.5 天）。

## 5. v1.4 可选清单

1. §4.3/§4.6：把“Stage 1 达量”改为有效 `n`+设计效应+`V` 不变性+污染审计四项出口（P0）。
2. §4.5/§4.6：新增 Stage 2 MDE/CI/稳健性/阴性结果出口表（P0）。
3. §2/§6/§8：将 B 独立 OSF 由补救升级为 B0 必做（P0）。
4. §4.2/§4.8：新增 trace 生命周期、权限和六个强字段（P0）。
5. §4.2：将“唯一干净手段”改为“主要可执行识别手段”，将 `V` 改为“候选充分统计量”（P0）。
6. §4.8：新增可观测性矩阵，覆盖行为漂移、任务污染和群体相关性（P1）。
7. §4.2/§4.7：探索臂最多一个，统一 `warning/hard_cap/owner`（P1）。
8. 页眉/§9：修正 v1.3/v1.4 版本漂移（P1）。

**结论：** v1.3 方向成立，不需重构；但目前是“B0 工程化、B1/B2 口号化”。在 P0 完成前不应放量 B1；P1 可随 pilot 实施；P2 不应阻塞首轮运行。
