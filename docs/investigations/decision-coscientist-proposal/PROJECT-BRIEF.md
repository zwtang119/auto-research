## 1. 项目摘要

**研究问题**：能不能让 AI 在电脑里把一场灾难「预演」几百上千次，用「打比赛 + 优胜劣汰」的方法自动找出最好的应急处置方案，并用真实灾难的官方档案检验：机器找出的方案靠不靠谱？

**一句话回答**：方法论上可行；新颖性经两篇最近邻全文精读 + 六条相邻文献线检索**未发现占位**；开题报告已通过两轮独立评审。唯一卡点是还没有跑出实验结果——下一步是一个 4–6 周的最小证据链试点（MVE），而不是继续改设计。

**方向名称**：Decision Co-Scientist（工作名）。把生物医学「AI 科学家」系统（Google Co-Scientist、FutureHouse Robin，2026 年 5 月同刊 Nature）的「假设生成→锦标赛排名→进化精炼→元评审」闭环，移植到 AI 辅助决策领域——载体为团队自研的决策推演系统 PolicySim，并用真实灾害记录（古雷石化爆炸、郑州暴雨）做「现实锚」校准。研究对象是一般性的高后果决策问题（应急处置、公共安全、城市与社会韧性等），灾害应急是首个试验场景。

**当前结论（终裁）**：GO（附条件）。首选投稿目标 NeurIPS Datasets & Benchmarks（锚数据集 + 锦标赛基准 + 校准协议三件套），IJCAI 2027 主会备选（AAAI 2027 时间过紧不列），Nature Computational Science 级别需一年以上多案例实证。

## 2. 背景

### 2.1 我们和我们的系统

本项目团队的方法论叫 **CDS（Computational Decision Space，计算决策空间）**，理念一句话：**让复杂决策先在机器中运行，再在现实中执行**。载体是 **PolicySim**：一个面向高后果决策的推演工作台（首个应用域为灾害应急）——用户描述决策问题后，系统用 LLM（大语言模型，底座为 DeepSeek-V4）驱动的多智能体角色扮演（现场指挥、消防、医疗、工程、环境、交通等八类角色）做三轮协商推演；同一场景用蒙特卡洛方法重复采样几十上百次，输出**六维后果指标的分布**（伤亡估计、控制时间、泡沫消耗量、次生灾害概率、环境影响范围、疏散完成率）而非单一答案，最后生成给指挥员看的决策报告。系统承担**国家重点研发计划课题五（灾害事故应急决策知识库）**的研究任务，知识库来自同计划课题三。

PolicySim 已有的能力：两个历史灾害场景（古雷石化、郑州地铁）+ 广西横州洪水（六蓝水库溃坝，2026-07）真实案例（晚于底座训练截止，天然防「背答案」；另有合成场景「企业洪水」仅用于开发调试、不进入研究）、知识库检索注入、AI 配置助手（「AI 只动嘴不插手」，人确认才运行）、人在回路。**缺的能力**：不会自动提出新方案、不会迭代改进、没有与真实灾害记录的定量对照、评估过度依赖 AI 评审且不稳定。本项目就是围绕这四个缺口展开的。

### 2.2 医学 AI「自动科研」浪潮（方法论的源头）

2025–2026 年，生物医学领域集中出现了一批「AI 科学家」系统——AI 不再只是辅助，而是端到端完成「提出假设→设计实验→分析数据→修正假设」的科研闭环：

- **Co-Scientist**（Google，*Nature* 2026）：七类专职 agent 组成闭环——生成假设、模拟同行评审、两两辩论打分排名（Elo 锦标赛，一种按两两胜负增减的评分制）、对头部假设做变异进化、元评审把评审经验回写到下一轮。在 203 个研究目标上，假设质量随算力投入单调上升；三个生物医学场景经实验室湿实验验证（发现 AML 药物重定位候选 KIRA6）。
- **Robin**（FutureHouse，*Nature* 2026）：文献 agent 与数据分析 agent 分工，打通「文献→假说→实验→分析」全闭环，发现干性黄斑变性新候选药 ripasudil（细胞实验证实）。全流程不到 2 小时、约 10.76 美元，对照人工估计 359–424 工时。
- **Biomni**（Stanford，bioRxiv 2025）：先从数万篇论文自动挖掘领域「行动空间」（150 工具/59 数据库），专家核验后构成统一 agent 环境；基准成绩与人类专家持平。
- **Claude for Science**（Anthropic 产品，2026）：每个产出绑定产生它的代码与环境，后台自动审查标记不可溯源的声明。
- **Elemento 的批评**（*Nature* 评论，2026）：目前 AI 的「发现」都是已有文献知识的重组而非原创——这条批评对决策领域同样成立，本项目把它的应对写进了设计（见 §4.4）。

### 2.3 问题的提出

科学发现和决策推演在抽象层是同一个搜索问题：**假设空间↔策略空间、湿实验↔蒙特卡洛推演、物理真值↔真实灾害记录**。既然医学能用「锦标赛+进化+真值锚」自动做科研，AI 辅助决策能不能用同一套方法自动找方案？

直接的困难有三个（这是医学系统不需要面对、而本项目必须解决的）：
1. **决策领域没有「湿实验」**——推演本身是 LLM 演的；PolicySim 项目开发记录中的实测显示，多次「独立采样」的输出方差为零（采样温度不构成统计独立随机性，目前靠提示词注入扰动参数构造方差）。
2. **裁判（LLM judge）有噪声且持久**——同一项目的实测显示，更换 judge 模型会使评估结论从「不确定」翻转为「强」。
3. **真值是回顾性的历史档案**——单点、混杂、有幸存者偏差（火灭了不等于处置最优）。

解决这三个困难所需的机制，构成本项目的核心贡献。

## 3. 文献综述（简版）

六条相邻工作线，每条只说最关键的几篇（完整 64 条见开题报告第 10 章）：

**① AI 自动科研系统**：Co-Scientist（DOI: 10.1038/s41586-026-10644-y）、Robin（10.1038/s41586-026-10652-y）、Biomni（bioRxiv 2025.05.30.656746）；唯一已知的跨域移植是 Minasny et al. 把 Co-Scientist 复刻到土壤科学（10.3389/fsci.2026.1721295）——证明「架构移植」可发表，但决策科学方向未见跟进。

**② LLM 社会/政策仿真**：Generative Agents（arXiv:2304.03442）奠基；AgentSociety（arXiv:2502.08691）、EconAgent（arXiv:2310.10436）等都不做策略搜索，验证普遍靠「复现社会现象」或定性对照。最近邻是 CMU 的 16 个月参与式设计研究（arXiv:2509.21868，13,000 agent 应急疏散）——但它明确拒绝优化范式、不做定量校准，与本项目正交互补。

**③ LLM 驱动进化/优化**：LEAR（GECCO 2025，DOI: 10.1145/3712255.3734368）用 LLM 进化 NetLogo 移动代码（适应度纯仿真内部）——「首次 LLM 进化用于多智能体」归它，我们不做此主张；EoH（arXiv:2401.02051）/FunSearch（Nature 2024）进化代码（适应度是可执行真值）；ADAS（arXiv:2408.08435）/AFlow（arXiv:2410.10762）/OPRO（arXiv:2309.03409）进化 agent 构造；DSPy（arXiv:2310.03714）/Promptbreeder（arXiv:2309.16797）优化提示参数。共同区别：**我们进化的是自然语言决策策略，评价是对真实灾害记录的校准度**。

**④ Self-play 理论（理论地基）**：PSRO（Lanctot et al., NeurIPS 2017, arXiv:1711.00832）确立「种群+两两对决+最优响应」框架；α-Rank（Omidshafiei et al., Sci. Rep. 2019）处理「石头剪刀布」式非传递排名（替代 Elo）；Gao et al.（ICML 2023）证明对带噪声奖励持续优化会过优化；Rad et al.（arXiv:2601.04411）给出相变判据 Youden J>0（判对率减判错率，大于零进化才有正期望）；Shao et al.（Spurious Rewards, arXiv:2506.10947）警示「随机奖励也能提升」的假象（故必须跨模型复现）。核心结论：**裁判质量决定进化天花板——judge 校准是理论必需品，不是工程细节**。

**⑤ ABM 验证（方法论谱系）**：POM（Grimm et al., Science 2005）用多个模式当模型过滤器；docking（Axtell et al., CMOT 1996）给随机模型比较的三级等价标准；Windrum et al.（JASSS 2007）把本方法归入「输出验证」；参数校准谱系（BehaviorSearch、Grazzini & Richiardi 2015）用「与数据的距离」标定数值参数——本项目与它的区分：搜索对象是**自然语言策略**、适应度是**历史外部真值**、驱动是**生成性闭环**（三重限定组合）；SLALOM（arXiv:2604.11466）的「停摆的钟」批评（终点对≠过程对）被本项目吸收为轨迹对照设计。

**⑥ 结算驱动校准**：以真实事件结算结果为真值已成预测评估范式（Halawi et al. 2024；Prophet Arena, arXiv:2510.17638），但只用于单点概率预测，无人用于多智能体推演。

**定位结论**：「Co-Scientist 式闭环 × 多智能体决策推演 × 真实灾害记录校准」这个三元交叉点上，**两篇最近邻全文精读 + 六条文献线检索均未发现占位**；但相邻各线都在快速生长——景观评估认为这个空白是时间敏感的（窗口期以月量级估）。

## 4. 研究方向与核心设计

### 4.1 一句话与三个比喻

让候选处置方案像**联赛**一样两两打比赛（各推演几十次比后果分布），像**育种**一样让赢家变异出下一代方案，再用真实灾害档案当**标准答案**对账。

### 4.2 怎么判断好不好？（四层评估）

**第一层：单场胜负——不用人。** 两个方案各推演 N 次，比较伤亡、控制时间等主指标的分布，统计判定；定性判断由校准过的 AI 评审（judge）打分，它头上有标准答案盯着（Gold 锚定：定期拿人事先写好答案的样例题抽查它有没有跑偏）。

**第二层：机制有没有用——基线 + 四个对照组。** 基线是 PolicySim 现行做法（AI 角色协商三轮直接出方案，下称「单批次协商」）；四个对照组是：随机多试几次挑最好的（best-of-N）、普通遗传算法（无比赛评审环节的「笨进化」）、人类专家亲自出的方案、直接问 AI「你觉得该怎么办」。同预算同尺子。只赢基线不算，赢了强对照才说明机制有增量。关键设计叫**锚分离**：锚的载体是「因子」（每个案例拆成几十条可检验事实），这些因子按类型分成两套不相交的池子——60% 在进化过程中当「教练」用（适应度锚），40% 冻结起来只在终末评估当「考官」用（评估锚），防止「自己出题自己考」。

**第三层：现实里对不对——跟真实档案对账。** 古雷/郑州官方调查报告拆成一条条可检验「因子」（泡沫吨数、控制时间、疏散率），检验方案是否**不违背已知证据与因果机制**（注意：不是要求复制历史——跟历史不同不扣分，违背物理闸门或证据支持的机制才扣分）。古雷是**正锚**（处置被证明基本有效、零死亡的成功案例），郑州是**负锚**（处置被官方认定失当的案例，检验系统避开失败链的能力）。**专家在这里出场**：≥3 名应急专家盲评（隐去地名与结局）。另有不用人的工程闸门：消防规范的泡沫量下限、疏散排队模型（NUREG/CR-7002，美国监管级方法）——它们不懂 AI，算出来多少就是多少。要坦白：闸门只能卡住明显离谱的输出，中间灰色地带仍靠 AI 推演——所以这一层只叫「旁证」。

**第四层：裁判可信吗——四道关。** ① 标准答案抽查（Gold 锚）；② 误判率远低于 10%（依据：一项 285B 参数的自我对弈实验显示 10% 随机误判即可抹平全部训练增益；另有理论相变分析要求 Youden J>0）；③ AI 评审排名与工程模型排名的 Kendall's τ（等级相关系数）≥0.5——这是本项目预注册的阈值；④ 换一个底座大模型，结论不能翻转。

**人到底在哪**（就三个位置）：① 人类专家出一套方案当「选手」（第三层对照组之一）；② 专家盲评当「历史裁判」（第三层对账）；③ 最终拍板永远是真人——系统底线原则「AI 只动嘴，不插手」。

### 4.3 四个假设（简版）

- **H1**：锦标赛+进化选出的方案在预注册主指标（伤亡估计、控制时间）上显著优于基线与全部对照组（赢现行做法不算，要赢强对照）。
- **H2**：方案质量随进化代数上升，且超过「封闭池里反复选最好」的伪进步曲线（选择伪影对照）。
- **H3**：冠军方案与 ≥2 个真实灾害记录中有效处置的证据**相容**（不违背已知证据与因果机制；判定规则：两案 ≥8 条裁定因子中 ≥60% 成立）——且必须显著优于「直接问 AI」的对照臂（排除「背答案」解释）。
- **H4**：AI 评审经校准后通过四道关（见 4.2 第四层）。

### 4.4 诚实边界（主动声明）

机器「进化」出的方案本质上是人类处置知识的重组（变异空间来自备案预案与人定选项池），我们主张的是**搜索效率与校准质量的提升**，不是机器原创处置方案；历史档案有幸存者偏差，对账结论只称「旁证」；后果分布一致不等于决策机制被验证；最终拍板永远是真人。

## 6. 关键结论

1. **映射成立**：医学 AI 自动科研的方法论骨架可移植到决策科学；三个断点（无湿实验、judge 噪声、历史真值）已转化为设计元素而非回避。
2. **新颖性未发现占位**：两篇最近邻全文精读 + 六条文献线检索均无；贡献收窄后站得住——决策策略语义层进化、三重限定组合的 calibration-as-fitness（自然语言策略 × 历史外部真值校准度 × 生成性闭环）、judge 噪声下稳健选择、正负双向真实锚。
3. **锚可建且已起步**：古雷 9 因子已编码（官方档案）；郑州负锚零成本就绪；锚池流水线（AI 发现与草编、专家核验与裁定、机器监测更新）+ 预案库行动空间矿线已设计。
4. **可行（附条件）**：唯一硬卡点是 **G2**（立项前检查项 gate 的编号，指「LLM 采样随机性的统计合法性」——多次采样方差为零的问题）。计划用拉丁超立方采样（一种分层均匀采样方法）+ 固定种子 + 模型版本钉死来解决；属 go/no-go 级，4–6 周内可证伪或解决。
5. **venue 定位**：NeurIPS Datasets & Benchmarks 首选（锚数据集+基准+协议三件套与该赛道匹配度最高）；IJCAI 2027 备选；PoliSim@CHI workshop 降级出口。

## 附录 B：术语表（按字母/拼音序）

| 术语 | 解释 |
|---|---|
| α-Rank | 不要求「A 赢 B、B 赢 C 则 A 必赢 C」的排名方法（策略常是石头剪刀布关系，Elo 会失真） |
| best-of-N | 随机采样 N 个方案、用同一评审挑最好的——对照组之一，排除「赢只是因为多试」 |
| CDS | 计算决策空间：把复杂决策变成可在电脑里模拟、比较、检验的形式（团队方法论） |
| Elo | 国际象棋式评分制：两两对决按胜负增减分数。本报告仅作展示 |
| Factor Ledger（因子账本） | 把灾害档案拆成一条条因子的账本：可观测指标、数值阈值、判定规则、反证信号、裁定状态 |
| G2 | 立项前检查项（gate）编号：LLM 采样随机性的统计合法性（多次采样方差为零的问题） |
| Gold 锚 | 人事先写好标准答案的样例题，用来抽查 AI 评审有没有跑偏（与「现实锚」是两个概念） |
| Kendall's τ | 等级相关系数：衡量两个排名是否一致（-1 到 1，越大越一致） |
| 蒙特卡洛推演 | 同一场景重复随机模拟几十上百次，得到后果的分布而非单一答案 |
| MVE | 最小证据链（minimal viable evidence）：用最小端到端实验验证机制可行性并实测成本 |
| 锚分离 | 因子分两套不相交的池子：60% 进化期当教练（适应度锚）、40% 冻结作终评考官（评估锚） |
| PSRO | 策略空间回应（Policy-Space Response Oracles）：「种群 + 两两对决 + 最优响应」的博弈论迭代框架，self-play 主线之一 |
| 预注册 | 实验前冻结假设、指标、判定规则与统计方法，防止事后挑数据 |
| 选择伪影 | 封闭池里反复选最好，分数机械性上涨——假进步，需先测出该曲线作对照 |
| 校验臂 | 不依赖 LLM 的独立工程参照（消防规范泡沫量、疏散排队模型），给推演卡物理闸门 |
| 现实锚 | 来自真实灾害档案的可检验事实（泡沫吨数、控制时间、疏散率），用于对账 |
| 拉丁超立方采样 | 一种分层均匀采样方法，比纯随机更均匀地覆盖参数空间 |
| Youden J | 判对率减判错率。J>0 才比瞎猜强；理论表明 J≤0 时进化不但无效反而有害 |
| 正锚/负锚 | 正锚=处置被证明有效的成功案例（古雷）；负锚=处置失当的案例（郑州），检验避开失败链的能力 |
