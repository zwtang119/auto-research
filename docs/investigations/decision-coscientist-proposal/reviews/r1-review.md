## 1. Novelty 区分度：3/5

**成立之处（应承认）**：
- 「Co-Scientist 式闭环 × 多智能体决策推演 × 真实灾害记录校准」三元交叉，经我对 LEAR 与 arXiv 2509.21868 **原文逐句对照**，确实均无占位：LEAR（GECCO '25 Companion, DOI 10.1145/3712255.3734368）进化的是 NetLogo 采集环境中 agent 的行为代码/伪代码，适应度为仿真内部资源计数，无外部真值锚；2509.21868（Li, Das, Shirado, 16 个月参与式设计，13,000 agent）明确拒绝预测/优化范式（§1："not as a set of outcomes to be optimized"），其 validation filter / enabling conditions 概念与报告转述一致。差异化落点（策略语义层进化、外部真值适应度、judge 噪声稳健选择、正负双向锚）的组合定位是清楚的。
- 对 ABM 验证谱系（Windrum 三进路 × POM × docking）的嫡系定位分析（§2.5）读得出是真精读过原文的，不是贴标签。
- 「首次 LLM 进化用于 MAS」的上位主张主动让予 LEAR（§2.3 末），分寸感正确。

**发现的问题**：

- **【严重 N1】「calibration-as-fitness 谱系中无先例」主张过宽（承重墙裂缝）。** §1.3 与 §2.5 称 ABM 谱系中"经验模式只被用作筛选器……从未作为生成性选择压力的连续适应度"。这不成立：ABM 参数校准的优化化进路早已把"与经验数据的距离"作为**被最小化的目标函数**——Stonedahl & Wilensky 的 BehaviorSearch（2010，NetLogo 生态中用 GA/进化搜索匹配 ABM 参数与目标数据）、simulated minimum distance 估计、Grazzini & Richiardi（2015, JEDC）对 ABM 的贝叶斯/SMD 估计、以及 ABC-for-ABM 文献。距离数据即 fitness，这与"筛选器"的差别只是连续化程度。报告真正能站住的主张是：**在 LLM 多智能体推演的策略语义层、以历史事件锚为外部真值、作为生成性选择压力**——这个收窄后的三重限定大概率仍为首例，但按现在的写法，任何熟悉 ABM 校准文献的评审人都可以一票击穿 novelty 论证。
- **【严重 N2】遗漏两条必引的最近邻文献线。** (a) 自动化 agent 系统设计搜索：ADAS（Hu et al. 2024）、AFlow（ICLR 2025）、EvoAgent（Yuan et al. 2024）、AgentSquare、GPTSwarm——它们进化/搜索的是 agent 配置与工作流（语义层），比 EoH/FunSearch/AlphaEvolve 的代码进化更接近本课题的"策略语义层进化"主张；(b) LLM 提示/策略自进化：OPRO（Yang et al. 2023）、Promptbreeder（Fernando et al. 2023）、EvoPrompt。§2.3 的"共同点"小结在这两条线下不成立。另：AlphaEvolve 在 §2.3 正文出现但参考文献（§10）无条目。
- **【严重 N3】遗漏 LLM-as-judge 偏倚与效度文献。** H4 整个论证以 judge 可信为前提，却未引 Zheng et al. 2023（MT-Bench/Chatbot Arena，位置/冗长/自偏好偏倚）、Panickssery et al. 2024（judge 偏好自家生成）、Taubenfeld et al. 2024（LLM 辩论模拟的系统性偏倚）、Wang et al. 2023（fairness）。Chatbot Arena 作为"大规模两两锦标赛 + Elo"的实证主场也未引；TrueSkill/Glicko 作为 Elo 替代谱系未讨论（α-Rank 论证见 S-维度，本身成立，但替代方案的排除需要交代）。
- **【严重 N4】对 LEAR 选择机制的表述与原文不符。** §2.3 称 LEAR"选择机制是 GA 亲本采样（非两两对决）"。LEAR 原文算法 1 第 14 行明确为 "Choose m parents using **tournament selection**"，§3.1.3 进一步描述"子集内按适应度比较、胜出者进入亲本池"。tournament selection 字面上就是锦标赛。"非两两对决"作为区分点不成立；真正的区分是"**LLM judge 的成对裁决** vs **适应度数值的锦标赛选择**"。此错误出现在宣称"逐项区分已全文核实"的段落，会直接损伤评审人对整条 novelty 论证的信任（另见 Hb2）。
- **【一般 N5】** LLM self-play 实证线未引：SPAG（Cheng et al. 2024）、Self-Rewarding LM（Yuan et al. 2024，模型自任 judge 的训练闭环——与 judge 噪声主题直接相关）、Absolute Zero（2025）。§2.4 目前只有理论与 RLVR 噪声文献。
- **【一般 N6】** Minasny et al. 的转述偏松：原文（Frontiers in Science, DOI 10.3389/fsci.2026.1721295）是用 **Manus AI 1.4** 通用 agent 模拟 Co-Scientist 式角色设计的 perspective + demo（5 个假设、无湿实验），并非对 Co-Scientist 本体的"复刻"。"目前唯一的跨域移植先例"宜软化为"最接近的移植先例"。

## 2. 证据链效度：3/5

**成立之处**：
- H1 的稻草人防御设计（"仅赢 C0 不构成支持，须赢 B1/B2 才归因为机制"）是同行评审会主动要求的对照，作者自己先写了，值得肯定。
- H2 的"选择伪影对照曲线"（封闭池重复选头部的机械性上涨先测作基线）是一个漂亮的、多数进化计算论文都缺失的对照。
- H3 的操作化颗粒度（锚类/分支类/反信号类分别预注册匹配粒度、两轮去标识化盲评、校准锚与检验锚分离）高于该领域平均水准；负锚（郑州 counter_signal）检验证伪能力的设计尤其好。
- 校验臂确为非 LLM 独立参照：GB 50151 规范下限、NUREG/CR-7002 ETE 排队模型、SFPE 水力下界闸门均为工程计算，判异规则（"推演泡沫 < 下限/3 且宣称扑灭→标红"）是诚实的"下限闸门"而非伪精确点估计。
- 污染控制五条（锚隔离、盲评、±30% 扰动平行变体、语料稀薄锚、泄露基线）具体可操作，且坦承两大锚都是超大公共事件。
- SLALOM 的 stopped clock 批评被正面吸收进设计（§5.4 轨迹保真对照），而非回避。

**发现的问题**：

- **【严重 E1｜条件性致命】H1 主指标的"优于"无评分规则，且评估锚集表述与锚分离存在循环风险。** "伤亡估计"作为预注册主指标（§5.5），"优于基线"指什么？更低？更接近锚？若"更接近锚"，则 H1 的评估度量与 C2 的适应度（现实锚校准度）同源；而 §5.2 又写明四条件三基线在"相同评估协议（同 judge、同扰动分布、**同锚集**）"下运行——若评估锚 = 校准锚，C2 胜出 H1 是**构造性的**（它直接优化了评估指标，基线没有），主结论循环、证据链归零。§5.5 的"校准用锚与检验用锚分离"目前只写在 H3 语境下，未声明适用于 H1 评估。**若作者本意是留出锚评估，必须显式写出锚分离方案覆盖 H1；若本意确为同锚评估，此项升级为致命（不解决则方向不成立）。**
- **【严重 E2】H3 缺少汇总判定规则，可证伪性不完整。** "与 ≥2 个真实灾害记录中事后判定有效的处置行动一致"——9 条古雷因子 + N 条郑州因子中须命中多少条、按何种加权、命中到什么程度算"一致"？无阈值则无论实验输出什么都能事后讲通。且同一事件内的因子彼此相关（同一份调查报告的编码），"≥2 个灾害"的有效独立证据量远小于表面——两个事件各取若干因子，本质上 n=2。
- **【严重 E3】校验臂覆盖面与 judge 效度论证错位。** H4 的效度腿是"judge 排名与校验臂参照的排序相关显著"，但校验臂只覆盖泡沫消耗与疏散两个物理量；主指标之一的**伤亡估计完全没有独立参照**，该维度的 judge 效度只能依赖 Gold 锚——而 Gold 锚本身出自同一批历史材料的人工编码，需交代编码者与锚使用的分离（编码人是否接触推演输出？），否则效度论证在该维度是 LLM↔人工编码的同源印证。
- **【严重 E4】生成侧与评审侧同源模型的共模噪声未讨论。** "更换 judge 模型不改变主结论"是必要的，但若生成器与 judge 共享训练分布（同底座家族），它们会共享盲区——跨 judge 稳健性会被高估，且与 N3 的自偏好文献直接相关。报告把"跨 ≥2 个底座复现"放在 H4 的"排除先验放大"名下，但同源 judge 问题未被命名。
- **【一般 E5】** B3 专家策略的生成条件未定义（专家是否见锚？前瞻还是回顾？若专家熟知古雷/郑州案情，B3 实际测的是"人类记忆"而非"人类设计能力"）；≥3 名专家做两轮盲评，样本对 κ 估计偏薄。
- **【一般 E6】** "逐局误判率 ≪10%"的阈值出处是 Chen（2026）AI 生成预印本的"实测阈值"——已披露其未验证，值得肯定；但作为预注册判据仍需独立出处或自行标定方案（如以 Gold 锚集上的实测误判率定阈）。
- **【一般 E7】** §2.1 引 Robin 的"LLM judge 重测一致性 88%（人类专家 61%）"作为信度≠效度的素材，该数字我未能独立核实（Nature 正文付费墙外的摘要未含），建议作者补可核查出处。

## 3. 统计严谨性：3/5

**成立之处**：
- 功效计算我复算无误：双侧 α=0.05、功效 0.8，δ=0.3 → 每组 n = 2(1.96+0.8416)²/0.09 ≈ 174.4 → 175 ✓；δ=0.2 → 392.5，t 精确解 ≈394–395 ✓。
- 放弃被本团队自我证伪的 N=240 先例、以 P12 实测反转教训（N=6 强信号 N=30 翻转）定 δ≤0.3 的诚实上限——这是方法论上正确的自我修正，比绝大多数功效分析实诚。
- 反转零假设 + bootstrap 的 Axtell et al. 1996 出处经原文核对**属实**（其脚注 13 明确提出 "reverse the usual null hypothesis formulation"、"differ by no more than X percent" 及 bootstrap 进路）；用于防小样本"未能拒绝"假阳性是对症的。
- α-Rank 替代 Elo 的论证（非传递博弈下 Elo 失真，引 spinning tops 几何与 Nash averaging）成立且与 self-play 文献衔接正确；Elo 仅作展示的降级处理恰当。
- 预算口径（总 token 含 judge/生成/元评审，各臂一致）定义干净。

**发现的问题**：

- **【严重 S1】分析单元与伪重复风险。** N=175–395 的一个"样本"是一次推演 run，变异来源是 LHS 场景参数 + 提示词扰动 + 种子。同一场景、同一 judge 下的 run 彼此相关（共 judge 噪声是持久性的，报告自己在 §1.1 承认）；配对设计解决跨臂比较，但臂内 i.i.d. 假设存疑。G2 把"输出分布的独立同分布性检验"列为门槛任务值得肯定，但预注册清单还应写明：若 i.i.d. 不成立时的备选方案（按场景种子分块的置换检验 / 层级模型 / cluster-robust 推断）。
- **【严重 S2】配对设计与检验选择不一致 + 等价界值未注册。** §5.5 同时写"优先采用配对设计"与"置换检验或 Mann-Whitney U"——配对设计应对应 Wilcoxon 符号秩或配对置换检验，MW-U 假定独立样本，二者矛盾。更关键的是：反转零假设的等价界值 X%（"差异不超过 X%"）**没有给出具体数值**——margin 不预注册，等价检验事后可任意调宽取值而必然"通过"，这正是该检验最常见的滥用形态。
- **【一般 S3】** 多重性校正族未定义：H1–H4 × 主/次指标 × 3 消融 × 2 场景，BH 在哪个 family 内做？主指标两个（伤亡估计、控制时间）之间是否再校正？
- **【一般 S4】** α-Rank 分数跨代追踪（H2 的"质量随代数上升"）在策略池扩张下不可比（新策略进入会重算全池稳态分布），需 league 式累计池或固定参考集的设计交代；α-Rank 在预期池规模下的计算成本未评估。
- **【一般 S5】** 等 token 预算下各臂实际 N 分配未给出：C1–C3 的 judge/进化/元评审开销会压缩其 sim run 数，各臂功效不等，B1（best-of-N）反而 run 数最多——需要一张按臂的预算拆分表。自适应配对的序贯窥视（peeking）校正缺失。
- **【一般 S6】** δ 的度量尺度未定：控制时间、伤亡估计通常重尾，标准化均值差定义在什么变换（log？秩？）上未说明。

## 4. 工程可行性：3/5

**成立之处**：
- 已有基础清单具体且声称经磁盘核查（679 行 tournament.py、约 1200 项自动化测试、古雷 9 因子试编码完成、cds4polymarket 校准库），零件复用面大，不是从零画饼。
- W4（G2 不过即终止）与 W12（锚协议未就绪即回退）双检查点 + 预写回退条款，是开题中少见的诚实治理结构；缺口清单（§6.2）如实到"连零件都没有"的颗粒度。
- 校验臂选型（GB 规范 + NUREG ETE + SFPE）有监管级方法与历史标定先例（Katrina/Irma、Goyang 2018 反演、480 起罐火案例库——Persson & Lönnermark 与 Kim et al. 2021 均核实存在）。

**发现的问题**：

- **【严重 F1】算力/成本预算缺位且核算时点倒挂。** 主实验规模：4 条件 × 3 基线 × 3 消融 × N≤395 × 2 场景 × ≥2 底座，叠加锦标赛两两配对（每场 = N 次双策略推演）与 judge/元评审 token，总量轻易达 10⁹ token 量级、数百至数千美元以上。"成本核算"却排在 W13–W17 主实验期间（§8）——等于先开枪后算账。成本/算力核算必须前移到 W4，与 G2 并列作为 go/no-go 门槛；若预算不支持 δ=0.2 的 N=395，应在开题阶段就声明只承诺 δ=0.3 的检验力。
- **【严重 F2】进度内部依赖链断裂。** §6.2 明说 sim 侧三个可观测性缺口（初控-复燃状态机、罐区 1.5D 几何、伤/亡分离）"不补齐，相关因子永远 inconclusive"——即它们是 G3 锚协议（W8–W12 定稿）的前置。但 §8 只在 W1–W4"启动"补齐，无完成检查点，W12 的锚协议检查点也未把 sim 缺口列为通过判据。依赖关系写了，门禁没接上。
- **【一般 F3】** W5–W9 五周内完成 M1–M4 四模块 + Gold 锚灾害域重建 + 郑州因子编码，即使站在现有零件上也偏乐观；全文无人力配置说明（几人？是否含课题五任务分摊？）。
- **【一般 F4】** 底座模型"版本钉死 + 快照"未说明快照形态（自托管权重 or provider 版本端点）；商用 API 在 16–20 周 + 投稿周期内退役/静默更新的风险没有预案；跨 ≥2 底座复现的成本未入预算（与 F1 联动）。
- **【一般 F5】** 专家 panel（≥3 名应急专家、两轮盲评）招募风险有承认但无降级预案（若仅 2 人到位，盲评设计与 κ 估计如何调整？）。

## 5. 诚实边界：4/5

**成立之处（本报告最强维度）**：
- Elemento 式重组声明（§7.1：主张限于搜索效率与校准质量，不主张机器原创处置方案）、幸存者偏差（§7.2）、复现≠解释（§7.3）、等终性（§7.4）、信度≠效度（§7.5）、单司法辖区（§7.8，附 Buncefield 跨辖区候选）——风险清单完整且每条附对策，高于该领域平均水准。
- Chen（2026）AI 生成预印本被显著标注"定理未经独立验证、仅作风格化参照"（§10 第 28 条）——披露到位。
- 内部方向切割（§7.7 与 Tsinghua ideas/09 的三轴切割 + 回退合并条款）透明。
- 参考文献声称"已经逐条核验"：我抽查 18 条（见下表），**无一伪造**，仅 2 处轻微瑕疵（Kim et al. 作者顺序待复核；Minasny 转述偏松）——该声明基本属实。

**发现的问题**：
