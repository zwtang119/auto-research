## 2. 命名（项目内术语）

- **现实锚（reality anchor）**：来自真实灾害档案的可检验事实（泡沫吨数、控制时间、疏散率）。
- **锚数据集 / Factor Ledger 锚账本**：锚的结构化集合。单条记录 = **因子**（factor），含可观测指标、数值阈值、判定规则、反证信号、来源分级、裁定状态。
- **Factor Ledger 结算协议（settlement protocol）**：把推演分布与锚逐条对照并裁定 supported / rejected / inconclusive 的机制。
- **适应度锚 vs 评估锚**：两套不相交的锚池（60%/40% 分层冻结），前者驱动进化，后者仅用于终末评估。
- **区分**：**Gold 锚**（Gold-H/M/L）是 judge 校准用的标准答案，与现实锚是两个概念，勿混。
- 文献谱系对应物：POM 的「模式过滤器」（Grimm 2005）、预测市场校准的「结算真值」（Prophet Arena）。

## 3. 建立方法（六步流水线，古雷已趟通）

1. **筛选**：三要素硬门槛（处置行动时序 + 后果量化锚 ≥3 个 + 官方调查档案公开可得）+ 2509.21868 的 enabling conditions（决策杠杆在权限内、可用运营语言讨论）。
2. **档案获取与来源分级**：official（官方调查报告）/ media（一手报道）/ eng（工程规范）三级前缀；口径冲突一律写区间（古雷泡沫 650/850/1467 吨的教训）。
3. **因子编码**：Factor Ledger 灾害域 fork schema；四类因子（前兆/抑制/分支/反证）俱全；双挂钩（推演可观测 + 历史可裁定）；锚写区间不写点。
4. **盲评裁定**：≥3 名专家、两轮制（首轮隐去地名与结局）、匹配粒度按因子类型预注册；inconclusive 率 <30% 正常、>70% 触发设计审查（沿用 rubric §6）。
5. **池分配**：按因子族分层 60/40 冻结（适应度池/评估池），实验前锁定。
6. **维护**：新证据走 amendment 协议（不改原条目，追加修订）；锚版本化。

单案成本（实测）：已有 seed 的场景 ≈1 人日（郑州）；需新建场景 ≈1.5–2 人日（天津港）；首批探针 ≈3 人日（古雷，含 schema 适配）。

## 4. AI 批量化：分层自动化架构

**原则：能自动的全自动，该留人的坚决留人**——锚的裁定独立性（rubric 的 adjudicator independence）是硬要求；AI 生成锚再让 AI 裁定，等于把全文反复批判的 LLM 自指问题从后门放回来。

| 层 | 内容 | 自动化程度 |
|---|---|---|
| **Tier 0 发现** | 从 Wikidata（SPARQL：灾害类型+死亡人数+日期）、Wikipedia 灾害列表、新闻流批量拉候选事件，预填基础字段（日期/地点/类型/伤亡） | **可 90%+ 自动**。注意 Wikipedia/Wikidata 是**发现级**来源：通常只有伤亡总数，没有处置时序与资源消耗，不能当锚级证据 |
| **Tier 1 档案定位** | AI 检索官方调查报告并验证可访问性。金矿源：中国应急管理部及各省应急厅调查报告、**美国 CSB（化工安全委员会，技术极详尽的公开报告）**、UK HSE、NTSB、EM-DAT（量化字段数据库） | **可基本自动**（本 pipeline 的 Q3/Q5 已由子代理演示：抓取+验证 URL） |
| **Tier 2 因子草编** | LLM 从报告全文抽取因子草稿（observable_proxy、quantified_threshold、settlement_rule、counter_signal），按 fork schema 预填 | **可自动出草稿**。LLM 读报告抽结构化字段是其强项 |
| **Tier 3 核验与裁定** | 人工抽查因子草稿（比例可随信任度调）、专家盲评、池分配冻结、修订批准 | **必须留人**。这是锚池可信度的根基，不可省 |
| **Tier 4 更新** | 新事件进入 watchlist → 监测官方报告发布（通常滞后数月）→ 自动触发 Tier 1–2 → 人工核验入库 | **可自动监测+人工入库**。横州六蓝是活例：事件 2026-07 发生，报告发布后自动提醒 |

**风险与对策**：
- Wikipedia 数字口径飘（编辑战、多版本）→ 只用于发现，锚值必须以官方档案为准并标来源级。
- 训练数据污染：锚事件大都在底座语料里——但污染影响的是推演侧（策略可能是背出来的），已有对策（B4 直接作答对照臂、±30% 扰动、语料稀薄锚优先）；锚侧只要求可核验，不要求保密。
- 批量质量漂移 → Tier 3 抽查率与 inconclusive 率监控联动，异常批次整批回炉。

## 5. 已有内部资产（不从零建）

- PolicySim `emergency-kb/`：灾害案例库（郑州报告全文等）+ OSINT 自动采集归档模块（Tier 0/1 的发动机）。
- cds-keyperson Source Ledger：来源可信度/新鲜度分级机制（Tier 1 分级可直接复用）。
- cds4polymarket Factor Ledger fork schema + 盲评 rubric（Tier 2/3 的格式与协议）。
- 横州六蓝 seed：零污染锚的现成载体（Tier 4 的首个监测对象）。

**一句话**：锚池可以建成「AI 发现与草编、专家核验与裁定、机器监测更新」的半自动流水线；全自动到 Tier 2 为止，Tier 3 留人是设计红线，不是效率妥协。

## 6. 医学文献对流水线的背书映射（范式可迁移性评估）

|---|---|---|---|
| Tier 2 AI 草编 → Tier 3 专家核验 | Biomni（action discovery agent 读 25 类目文献挖行动空间 → 专家核验 + 测试用例） | 「AI 挖矿、专家验收」的组织方式；放行标准 = 自动产物匹配专家水平（74.4% vs 74.7%）才扩大自动化 | **强** |
| Tier 2 防幻觉 | Co-Scientist Reflection（接外部检索核验新颖性/可信性，有消融证据）；Claude Science（全程 provenance + 后台 reviewer 标记不可溯源声明） | 因子草稿必须回指报告原文章节；未溯源声明自动标红 | **强** |

## 7. 第二条矿线：应急预案库与行动空间挖掘

**与锚池的区分（勿放错位置）**：预案库是**规范性**来源（应该怎么做），挖的是**行动空间**（干预维度、选项池、资源基线、处置规程），进 M1 假设生成 / M3 进化变异空间 / M4 评审合规判据；事故调查报告是**描述性**来源（实际发生了什么+后果），挖的才是**现实锚**。预案不能当锚用。

**可得性事实（2026-07-19 核查）**：法律框架完整（国办《突发事件应急预案管理办法》2024 修订、应急管理部 2 号令；五级预案体系）；应急管理部官网有预案频道（mem.gov.cn/fw/yjya/）；**但无统一公开国家级预案库**，预案分散于 mem.gov.cn、各省政府网、企业公示与民间汇编——聚合成机器可读语料库本身即有价值的构建物。企业预案多不公开，W1 需核实可得范围。

**增益**：
1. 补上手工选项池缺口——选项池可扩展且每个选项回指备案预案条款（可审计）；Elemento 边界变为可声明设计：「变异空间来自备案预案库」。
2. judge 校准的 Gold 锚素材——预案合规性构造已知答案评审题（违反某预案第 X 条 = Gold-L）。
3. 「预案 vs 实战」gap 分析作为论文亮点小节（现成证据：郑州挡水墙设计 0.24m vs 验算 0.5m、村级预案擅删 200 人转移名单）。

**范式背书**：这是 Biomni 范式的直接对应物（挖专业文献建行动空间），背书强度远高于 Wikipedia 挖掘（专业文献、高信噪比、有验收先例）。

**风险**：预案质量参差（形式主义/抄袭/模板化普遍）→ 来源分级（国家级 > 省级 > 企业；有实战或演练修订记录者优先）；预案 ≠ 实战，仅作行动空间先验，不作真值。
