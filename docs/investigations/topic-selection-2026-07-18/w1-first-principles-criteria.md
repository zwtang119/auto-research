## 1. 原文中所有与"选题/什么问题值得研究/论文定位"相关的原则（按 L1/L2/L3 分层）

> **约定**：每条原则给出 (a) 提炼出的选题含义，(b) 该含义的原文引文，(c) 该条所来自的层级（L1 = Gewin 2018 原文；L2 = Horejs 2021 webinar；L3 = 项目 2026-07-17 并入），(d) 一句诚实判断：原文是"直接给选题准则"还是"写作建议被项目升维为选题准则"。

### 1.1 L1 原文（Gewin 2018）—— 写作建议中的选题暗线

原文 §1–§6 没有任何一条以"如何选题"为主语。但有六条写作建议**暗含选题约束**，可被升维为"选题时若不满足，则后期写作救不回来"的判据：

**L1-P1 (Mensh §2)：核心信息必须能用一句话表达，且该句就是标题。**
- 原文引文："论文应该重点传递一条核心信息，而且要反映在论文**题目**里面。不仅如此，论文中的所有内容都应该在逻辑上和结构上支撑该观点。"
- 选题含义：**选题尚未聚焦的标志是——你写不出一个能传达核心信息的题目**。如果一个方向用三句话还讲不清它在做什么、做出了什么，该方向就还没有到"可被考虑为 first-class paper 候选"的阶段。
- 引文辅证："先懂规则，再谈创造"——选题阶段同理：先有一个清晰的"题目句"，再去谈创新。
- 诚实判断：**写作建议被升维为选题判据**。原文未直接讲"选题时该做什么"，但若一篇文章必须用一句话写成标题，则反过来意味着"不能用一句话写成标题的方向 = 题目都没想清楚 = 不该选"。

**L1-P2 (Murphy §3)：红线 = "新颖出彩处何在？" (rote Faden)**
- 原文引文："在科学研究中，这条红线即'**新颖出彩处何在？**'它是撰写论文的全部原因。一旦确定了这一点，论文就围绕这条红线构建逻辑单元，展开论述。"
- 原文引文："**不要防御性写作**……如果以防御的姿态写论文，论文就会变成含糊的'散文'。"
- 选题含义：**若一个方向写不出"新颖出彩处何在"的一句话回答，它就没有红线 = 写作的全部理由都不存在**。"防御性 framing" 的反面是"用一句话答出新且引人入胜之处"——若答不出，方向就废了。
- 诚实判断：**写作建议被升维为选题判据**。"红线的存在性"在选题阶段就该被问——不是写完再问。

**L1-P3 (Borja §1)：讨论必须把研究放在整个研究大背景下，证明结果为何重要或具原创性。**
- 原文引文："大量论文被拒是因为'讨论'部分过于薄弱，明显表明作者并不十分了解已有的文献。作者应该将自己的研究结果置于整个研究大背景下讨论，证明这些结果为何具有重要意义或原创性。"
- 选题含义：**选题时必须问：能否在"大背景"里讲清本工作的"重要意义或原创性"？讲不清 = 方向太窄或太时髦而无根基**。
- 诚实判断：**写作建议被升维为选题判据**。原文讲的是"讨论部分要厚实"，但隐含命题是"若选了一个不与大背景接轨的题，讨论就厚不起来"。

**L1-P4 (Konkiel §6)：面向广大读者，标题对非专业读者友好。**
- 原文引文："**清晰的标题 + 非专业语言 = 跨学科影响力 + 公众可理解**……**面向广读者写，是放大影响力的杠杆**。"
- 选题含义：**选题时若只能写一个堆满术语的标题、或者只有自己 immediate field 能看懂，那该方向就不具备"广读者"潜力 = 影响力天花板已被方向本身决定**。
- 诚实判断：**写作建议被升维为选题判据**。原文讲的是"如何写标题"，但"能被跨学科读者看懂"本质是方向属性，不是文笔问题。

**L1-P5 (Gorsuch §5)：主张与证据必须一致；讨论必须解释发现的意义并考虑所有可能的解释。**
- 原文引文："保持论文主张与证据一致……确保考虑了所有可能的解释。"
- 选题含义：**选题时必须先问：手头的数据 / 方法 / 工具能否支撑一个值得讨论的"主张"？若答案是否——"我们做了一个实验"本身不是主张——该方向应被砍**。
- 诚实判断：**写作建议被升维为选题判据**。

**L1-P6 (Mensh §2 + Konkiel §6 共同暗线）：引导"一无所知"的读者。**
- 原文引文："**除非你告诉了我为什么要关注你的实验，不然我是不会知道的。**"
- 选题含义：**选题时若答不出"为什么外行要关注" = 没有 why-to-care = 方向没有跨域钩子**。
- 诚实判断：**写作建议被升维为选题判据**。

**L1 总结**：Gewin 2018 原文 **未给"选题准则"**——它给的是"如果选题通过这些测试，写作才会顺"的暗线。把它们反向用为"选题过滤器"是 **项目 2026-07-17 的升维操作**（中译 §7 段头 ⚠️ 自承），不是原文作者 Gewin 的本意。

### 1.2 L2 补料（Horejs 2021 webinar）—— 唯一一处直接讲"选题"

**L2-P0（Horejs 视频开篇，最简元原则）**：
- 引文（来自中译 §7 段头）："**Great research. You cannot turn crappy research into a great paper.**"（伟大的研究。你无法把糟糕的研究变成一篇伟大的论文。）
- 选题含义：**写作好不能补救研究差**。所有后续选题准则都是这条元原则的展开。
- 诚实判断：**直接给选题元原则**。Horejs 是 Nature Reviews Materials 高级编辑，这句话是她作为编辑对投稿的元判断。

**L2-P1 (Q1)：研究问题对多个领域重要，不只对你 immediate field。**
- 引文（来自中译 §7 段头）："1. **The research question needs to be important for many fields, not just for your immediate field.**"
- 选题含义：见原引文。
- 诚实判断：**直接给选题准则**。这是 Horejs webinar 结尾的总结性结论之一（不是正文引申），与 L1-P4 (Konkiel) 互补——Konkiel 讲"如何写"广读者，Horejs 把它升维为"问题本身对多领域重要"。

**L2-P2 (Q2)：数据必须鲁棒（可重复 / 有外部锚 / 有 ablation / 跨模型 / 跨 benchmark）。**
- 引文（来自中译 §7 段头）："2. The data needs to be robust."
- 选题含义：见原引文。
- 诚实判断：**直接给选题准则**。与 L1-P5 (Gorsuch 主张与证据一致) 同源，但 Horejs 把它升维为"数据不鲁棒 = 题不该选"，而非"写作时注意一致"。

**L2-P3 (Q3)：发现对他人有相关性（能被接住、能推动未来工作）。**
- 引文（来自中译 §7 段头）："3. **The findings need to be of relevance.**"
- 选题含义：见原引文。
- 诚实判断：**直接给选题准则**。Horejs 给了"relevance"一词但未在 webinar 结尾展开——具体展开（"能被他人接住、推动未来工作"）是**中译文件 §7.1 表格的展开**，不是 Horejs 原话。

**L2-P4 (D-E-F 标题公式)**：
- 引文（来自中译 §7.2）："D — Declarative (陈述式)…E — Engaging (引人)…F — Focused (聚焦)"
- 选题含义：选题的呈现层。一个 first-class 候选方向**应该能用 D-E-F 标题写出**——写不出 = 标题侧的 first-class 性缺失。
- 诚实判断：**直接给标题准则**。Horejs 给出，但原 webinar 文本未被中译逐字引用，本节是中译的转述。

**L2-P5（紫橙九句摘要公式）**：
- 引文（来自中译 §7.3 表格）："句 1 红（外行也看得懂）…句 2–3 红（背景扩展）…句 4–5 绿（一般问题）…句 6–7 紫（主要结果）…句 8–9 橙（更广语境）"
- 选题含义：摘要的"紫橙倒置"（把背景写满、结果挤没）是最常见错误——但**这是摘要写作错误，不是选题错误**。
- 诚实判断：**直接给摘要公式，但与选题无直接关系**。本条应归入"写作侧"而非"选题侧"——昨日 recalibration 把 PIT-703 列为"摘要紫橙倒置"是正确的归类（与 PIT-701/702 并列但属于不同子层）。

**L2 总结**：Horejs 2021 是**唯一**直接给出选题元原则与三准则的来源——三准则 (Q1/Q2/Q3) 即中译 §7.1 表格的全部内容；§7.2 D-E-F 与 §7.3 紫橙九句是**写作侧准则**，不应与选题准则混用。

### 1.3 L3 项目合成（2026-07-17）—— 把 L1+L2 压成"一句话"与新增反模式

**L3-P1（一句话元判断）**：
- 引文（来自中译 §7.4）："**一个 first-class paper 的选题，必须能在一句话里同时回答三个问题——'对谁重要？'（Q1 广读者）、'凭什么相信？'（Q2 鲁棒数据）、'接下来谁接得住？'（Q3 相关性）。三个回答缺一个，写作再好也救不回来。**"
- 诚实判断：**项目合成**。把 Horejs Q1/Q2/Q3 用"一句话"形态表达，是项目 2026-07-17 的句式包装，不是 Horejs 原话。"写作再好也救不回来"复述 Horejs L2-P0，但**"一句话里同时回答三个问题"**是项目发明的句式结构。

**L3-P2（§8 表格中标"新增 PIT-700 系列"）**：
- PIT-701 选题三缺一：项目发明。引文："选题只满足 Q1（重要）但缺 Q2（数据不鲁棒）或 Q3（无广读者）就动笔。"
- PIT-702 标题缺 D/E/F：项目发明，是对 L2-P4 的反模式化包装。
- PIT-703 摘要紫橙倒置：项目发明，是对 L2-P5 的反模式化包装。
- 诚实判断：**项目发明**。中译 §8 段头自承"新增反模式（建议合并入 experiment-pitfalls.md）"——这三条是**项目在原文与 Horejs 之外自行命名的反模式**，不来自任何源。

**L3-P3（§8 既有标尺映射）**：
- 引文（来自中译 §8 表格）："exemplars B1(深度)+B3(外部锚)+B7(作者/广读者) 的方法论版重述"
- 诚实判断：**项目合成**。中译 §8 表格把 L1-P3、B1、B3、B7 串成一行映射，是项目 2026-07-17 把新准则并入既有 exemplar 7-criterion bar 的对接方案。

## 2. 由这些原则推出的可操作选题检验清单

> **结构**：每条 (a) 检验问题（中文，可对任何方向直接问），(b) 推导依据（引 §1 中的哪一条），(c) 如何验证（具体动作 / 可量化判据），(d) 该条所来自的层级（决定它是"权威"还是"项目内部"）。

### C1. 一句话标题检验

- **问题**：能否用一句 ≤30 词的句子把该方向的核心信息写成（暂定）标题？该句是否反映"做了什么、为什么重要"两件事？
- **依据**：L1-P1 (Mensh) + L1-P2 (Murphy 红线) + L2-P4 (D-E-F)。
- **如何验证**：
  1. 强制作业：关掉所有笔记、参考文献、讨论记录，**用 1 分钟写出标题句**。
  2. 标题句 = "[主语做了什么] + [为什么这件事对 ≥1 个具体群体重要]"——若写不出第二段，"为什么重要"就是模糊的。
  3. 把标题句给 ≥2 个**非本领域**的人看（30 秒内），问"这在讲什么"——若他们答不出，本方向在标题层就缺广读者潜力。
- **失败信号**：标题用"我们做了 X"或"X 的研究"或"一种 Y 方法"开头而无"为什么重要"——这是 L1-P1 失败，是项目合成"红线缺位"的标志。

### C2. 红线一句话检验

- **问题**：能否用一句 ≤25 词的句子回答"**新颖出彩处何在？**"（rote Faden / what's new and compelling）？该句是**方法学发现**而非**数据独有性**？
- **依据**：L1-P2 (Murphy) + L2-P2 (Q2) + L3-P1。
- **如何验证**：
  1. 强制作业：写下"新颖出彩处何在？"的答案。
  2. **反向生存测试**（来自 F4 7-check test Check 2 / 项目 D17 规则）：若把答案里的"我们独占的数据/场景"删掉，红线句是否仍成立？若答案 = "我们做了 X 数据集的研究"——删数据 = 删论文 = 失败。
  3. 答案的谓语应该是"提出 / 证明 / 揭示 / 解构 / 重建"（方法学动词），不应是"采集 / 构建 / 报道"（数据 / 描述动词）。
- **失败信号**：红线句 = "我们首次构建了…" / "我们首次报告了…"——这是 L1-P2 "防御性 framing" 的标志，**项目 2026-07-17 PIT-701 的典型场景**。

### C3. 大背景 / 为什么重要检验

- **问题**：能否用 ≤150 词把该方向置于"整个研究大背景"中讨论：别人已经做了什么、还缺什么、本工作补了哪一块？是否能在"补哪一块"这一步**显式列出 ≥1 个已有文献的引用**（证明了解已有文献）？
- **依据**：L1-P3 (Borja) + L1-P4 (Konkiel)。
- **如何验证**：
  1. 强制作业：写 ≤150 字的"研究背景—缺口—本工作定位"三段。
  2. 三段中**必须出现至少 1 个 prior-art 引用**（arXiv / 期刊 / 顶会）证明你了解缺口在文献中真实存在。
  3. **同形词风险检验**（来自 F3 Finding[9]：Hyndman "reconciliation" 与本项目 "settlement reconciliation" 是同形 false cognate）：本工作用的核心术语是否在文献中已存在**含义不同**的用法？若是，必须在背景段显式 disambiguate。
- **失败信号**：背景段只引用本项目既有工作 / 自媒体 / 自造术语——这是 L1-P3 "并不十分了解已有的文献" 的标志。

### C4. 广读者 / 跨域重要性检验

- **问题**：能否用 ≤30 词向一位**非本 immediate field**的研究者讲清"这件事跟你有什么关系"？能否在"为什么对你有关"一步列出至少一个**非本领域**的具体应用场景？
- **依据**：L1-P4 (Konkiel) + L1-P6 (Mensh) + L2-P1 (Q1)。
- **如何验证**：
  1. 强制作业：邀请一位**本 immediate field 之外**的同行（建议 ≥1 个学科距离，如 NLP → 统计物理 / 应急决策 / 临床医学），3 分钟内口头讲清本方向。
  2. **30 秒电梯测试**：30 秒内必须说出"对 XX 领域的 XX 问题有用"——说"对所有领域都重要"是空话。
  3. 跨域钩子必须是**实质机制**（"X 领域的 Y 问题用了本工作的 Z 方法学可被解构"），不是表面类比（"X 领域也用 AI 哦"）。
- **失败信号**：跨域钩子是"我们可应用到 X / Y / Z 领域"——这是 L1-P4 + L2-P1 同时失败的标志，**项目 2026-07-17 PIT-701 的 Q3 缺位**典型。

### C5. 数据鲁棒 / 主张-证据一致检验

- **问题**：当前手头的数据 / 方法 / 工具能否支撑一个**值得讨论的"主张"**？该主张是否有 ≥2 个独立证据（外部锚 / 跨模型 / 跨 benchmark / ablation）？若 reviewer 提出"主张证据不足"——能否在不补新数据的前提下回应？
- **依据**：L1-P5 (Gorsuch) + L2-P2 (Q2)。
- **如何验证**：
  1. 强制作业：写下"本工作的可被验证主张"清单，每条标"证据来源"。
  2. **主张—证据矩阵**：每条主张必须有 ≥2 个独立证据；任何只有 1 个证据的主张都标"待补"（若 4 周内补不到 = 该主张应被砍）。
  3. **替代解释审视**（L1-P5 原文："确保考虑了所有可能的解释"）：对每条主张，写下 ≥2 个替代解释并说明为何本工作的证据排除了它们。
  4. **可复现性测试**（F4 Check 7）：他人能否用公开 / 易采集数据复现本方法学洞见？若必须用本项目独占数据 = Q2 不达标。
- **失败信号**：主张 = "我们观察到 X" 且证据 = "在我们的数据上观察到 X"——这是 L1-P5 + L2-P2 同时失败的标志，**项目 2026-07-17 PIT-701 的 Q2 缺位**典型。

### C6. "能否被他人接住" / 未来工作钩子检验

- **问题**：本工作产生的**方法 / 框架 / 数据 / 失败模式**能否被**不是本组的他组**直接拿去继续做？读者读完后能否写下"那我接下来可以……"的一句话？该方向**是否存在他组可立即 follow-up 的具体下一步**？
- **依据**：L2-P3 (Q3) + L1-P3 (Borja "用一两句话说明未来计划")。
- **如何验证**：
  1. 强制作业：写 ≤50 字描述"他组可立即 follow-up 的具体下一步"。若写不出具体动作（"探索更多应用"），本方向就是工具独占型或数据独占型——**违反项目 D-D14 规则**。
  2. **失败模式签名**（F4 Finding[6,8]）：是否在主张 / 引言中依赖"我们独有的数据"？若是 = 移除数据独有性后主张崩塌 = 触发 NeurIPS / ACL 数据独有拒信模板。
  3. **讨论—结论前瞻检验**（L1-P1 Borja 原文："用一两句话说明未来计划"）：结论的"未来计划"是否具体到"他组可在本方法上 follow-up"的程度？
- **失败信号**：未来计划 = "进一步研究 X 的其他方面" / "扩展到更多场景"——这是 L2-P3 失败，**项目 2026-07-17 PIT-701 的 Q3 缺位**典型。

### C7. "不能被数据独有救活" 的抗压检验

- **问题**：若所有 reviewer 一致认为"数据 / 方法 / 评估有 deep concern"，**仅靠"我们数据是独有的"能否救活**？答案若为"不能"——本方向不是被数据独有救活的题（✓ 通过），是被方法学支撑的题。
- **依据**：L1-P5 (Gorsuch) + L2-P2 (Q2) + F4 Finding[8]（NeurIPS AC 原文："authors will not succeed by pointing out additional strengths if the deep concerns are true"）。
- **如何验证**：
  1. 反事实作业：假装 reviewer 提出"你的方法学有 X 漏洞"，你能否在不提"我们数据独有"的情况下回应？若不能，本方向 = 数据驱动赌博。
  2. **F4 7-check test**（来自 F4 §Test）：逐条跑 Check 1–7，至少 Check 1 / 2 / 5 / 7 必须 PASS。
- **失败信号**：本方向核心价值的论证段落里 ≥3 次出现"我们的数据集" / "我们独占的数据" / "first-of-its-kind dataset"——这是 L2-P2 失败，**项目 D17 规则 + F4 Check 2 同时失败**。

### C8. 摘要紫橙 / 标题 D-E-F 形态检验（写作侧，非选题侧）

- **问题**：本方向的**暂定标题**能否通过 D-E-F（陈述式 + 引人 + 聚焦）？**暂定摘要**能否在 6–7 句就完成"主要结果"（紫句）？在 8–9 句完成"更广语境"（橙句）？
- **依据**：L2-P4 + L2-P5。
- **如何验证**：
  1. **D-E-F 标题 3 问**：陈述式（关键发现出现在标题）？引人（不堆术语、外行能查词典读完）？聚焦（明确你工作所在领域）？3 项中任何 1 项缺位 = PIT-702。
  2. **紫橙九句摘要**：按 §7.3 表格句序排 9 句。**最常见错误**：紫句被背景句挤没 → PIT-703。
- **诚实边界**：C8 是**写作侧**，**不是选题侧**——一条方向即便 C1–C7 全通过，若 C8 失败，paper 仍能写好（反之亦然）。把它列为 C8 是为了在"通过 C1–C7 后再过最后一道"，不是把摘要准则混入选题。
- **失败信号**：标题全是"X 领域 Y 方法的研究" / 摘要前 5 句是背景、紫句在第 7 句之后——这是 L2-P4 + L2-P5 同时失败。
