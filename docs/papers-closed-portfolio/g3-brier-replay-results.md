## 0. Sandbox workaround

Python sandbox blocks `os.walk`, `Path.rglob`, `os.scandir`, and `subprocess.find` from enumerating cds4worldcup files (returns 0). Direct `open(file)` and `Path.exists()` work on known paths. This v3 script uses a **hardcoded file list** of 2 known settlements + 6 known prediction cards from the user's post-gate report.

## 1. Method

1. Hardcode 2 settlement paths: wc2026-a-m01-mex-rsa (plan-c) + wc2022-g-arg-ksa (fixtures/cds4polymarket).
2. Hardcode 6 prediction card paths: 2 for wc2026-a-m01-mex-rsa (v0.1 + v0.2), 1 for wc2022-g-arg-ksa, 3 for plan-c matches.
3. For each (match_id, settlement) pair, for each candidate card, recompute Brier / log_loss from card probs vs settlement observed outcome.
4. Pick the card whose recomputed Brier is closest to the settlement's reported Brier (tolerance: Brier within 0.001, log_loss within 0.005).
5. Emit verdict: MATCH (unique) / MULTIPLE_MATCH / MISMATCH / NO_CARD.
