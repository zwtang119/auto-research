# 深度视觉审查 Prompt — CDS4WorldCup 公开站点

> 使用方式：将此 prompt 整段发给 Mimo Code，它会截图、审查、输出报告。
> 适用时机：修复 P0/P1 后、重大改动后、部署前。

---

## 任务

你是一个视觉审查员。你的任务是对 CDS4WorldCup 公开站点做一次全面的视觉审查，找出所有可见的布局、内容、合规和体验问题。

你需要：
1. 启动本地服务器
2. 对 6 个页面各截 2 张图（桌面端 + 移动端）
3. 用你的视觉识别能力逐张分析
4. 输出一份结构化审查报告

---

## 第二阶段：逐页视觉分析

对每一张截图，用你的眼睛回答以下问题。**不要读代码，只看截图。**

### A. 布局检查

1. **桌面端是否为多列布局？**
   - 首页 Hero 区应该是左右双列（文字 + 数据面板）
   - 球队卡片网格应该是 3-4 列
   - 详情页的信息区应该是多列
   - 如果全部是单列，那就是 P0 CSS 回归

2. **移动端是否为单列布局？**
   - 所有网格应该折叠为 1 列（个别可以为 2 列）
   - 文字不应该溢出屏幕右侧

3. **有没有横向滚动条？**
   - 任何页面出现横向滚动 = bug

4. **导航栏是否正常？**
   - 桌面端：能看到 "CDS4WorldCup" logo + 导航链接
   - 移动端：导航不应该折行或溢出

### B. 内容检查

5. **中文是否正常渲染？**
   - 没有乱码（方块字、问号、菱形）
   - 没有意外的英文长句（少量英文术语如 xG、FIFA 可以）
   - `text-transform: uppercase` 对中文没有造成异常效果

6. **国旗 emoji 是否显示？**
   - 球队卡片上的国旗 emoji（🇦🇷 🇫🇷 🇪🇸 等）是否可见
   - 如果显示为方块或空白 = bug

7. **概率数字是否合理？**
   - 所有百分比应该在 0%–100% 之间
   - 如果出现负数、>100%、NaN、undefined = bug

### C. 图表检查

8. **首页"夺冠难点分布"条形图**
   - 应该显示 9 个水平条（太依赖小比分、心理压力、签表太硬 等）
   - 条形长度应该和数字成比例
   - 条形应该有颜色（不是空白）

9. **首页"球队 × 难点"热力图**
   - 应该是一个表格/矩阵
   - 颜色深浅应该表示强度

10. **首页"外部参考 Top 8"图表**
    - 标题写的是什么？"AI 模型群体 vs 市场参考"？
    - 实际显示了几组数据条？只有一组还是两组对比？
    - 如果标题说"vs"但只有一组 = P1 bug

11. **详情页障碍条形图**
    - 阿根廷/西班牙的详情页应该有一个"夺冠难点"图表
    - 条形是否可见？

12. **详情页外部参考区**
    - 是否显示了概率数据？
    - 来源标签（可靠事实/待核验线索/只能参考）是否可见？

### D. AI 多视角模块检查（首页 + 详情页）

13. **首页 "AI 多视角推演" 区域**
    - 是否存在一个叫 "300 个视角，不是一个声音" 的区域？
    - 是否显示 10 个派别卡片（数据派、赔率派 等）？
    - 每个卡片内是否有彩色小圆点（点阵）？
    - 颜色是否为莫兰迪色系（灰蓝、赭红、橄榄绿、暖金、灰紫……），不是鲜艳的原色？

14. **详情页 AI 模块**
    - 阿根廷/西班牙详情页是否有 "AI 多视角怎么看这队？" 模块？
    - 是否显示了 2-4 个派别的分析片段？
    - 是否有"只能参考"标签？

### E. 合规检查（最重要）

15. **品牌泄露扫描**
    在截图的可见文字中搜索以下词汇。它们**绝对不应该出现**：

    - Kimi
    - 小米
    - Xiaomi
    - MiMo
    - MiMo Code
    - Long-Horizon

    如果出现任何一个 = **P0 合规违规**

16. **投注语言扫描**
    在截图的可见文字中搜索以下词汇。它们**绝对不应该出现**：

    - 投注建议
    - ROI
    - PnL
    - Sharpe
    - Kelly
    - 仓位
    - 低估
    - 高估
    - 正期望
    - value bet

    如果出现任何一个 = **P1 合规违规**

17. **开发者内容扫描**
    以下内容**不应该出现在访客可见的页面**：

    - 文件路径（`scripts/`, `data/processed/`, `artifacts/`）
    - "后续更新"
    - "不用重写页面"
    - Python 命令（`python3 scripts/...`）

### F. 一致性检查

18. **导航栏一致性**
    - 所有页面的顶部导航是否风格统一？
    - logo 链接是否都指向首页？

19. **配色一致性**
    - 背景色是否统一为暖纸色（`#FBF9F1` 或接近的米白）？
    - 文字是否统一为深墨色（`#13223A` 或接近的深蓝黑）？
    - 有没有突然出现白色背景或黑色背景的区域？

20. **来源标签一致性**
    - "可靠事实"、"待核验线索"、"只能参考" 这三个词在所有页面中使用是否一致？
    - 有没有页面用了其他表述（如"绿色来源"、"一级来源"）？

## 第三阶段：数据对照抽查

从 `site/data/homepage.json` 或 `site/data/baselines.json` 中读取 2-3 个数据点，对照截图：

1. **障碍条形图**：读 homepage.json 中 obstacle_distribution 的数字，对比截图中条形长度是否大致匹配
2. **外部参考概率**：读 public_model_crowd 中 top_teams 的概率值，对比截图中显示的百分比
3. **球队卡片数量**：首页应该展示前 8 支热门队的卡片，数一下截图中有几个
