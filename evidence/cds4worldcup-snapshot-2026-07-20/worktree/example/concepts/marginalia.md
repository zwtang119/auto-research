# Marginalia 边注系统

> **定义**：一种文档级批注记忆系统，将记忆以批注形式附着于知识点，实现"同位、无损、解耦"三大核心特征。
>
> **学术隐喻**：借用书籍"页边批注"的经典学术传统——在知识点旁边直接写批注，下次读到这行，批注就在那里。

---

## 三大核心特征

### 1. 同位（Co-located）

记忆与知识点同位置，读知识点即读记忆。

**vs 传统方式**：传统需要额外检索步骤（查询数据库 → 检索记忆 → 返回结果）

### 2. 无损（Lossless）

批注原文保存，不经过 Embedding 压缩。

**vs 传统方式**：向量检索有信息损失（768 维向量无法保留完整上下文）

### 3. 解耦（Decoupled）

记忆独立于 LLM/Agent/Session，可迁移。

**vs 传统方式**：传统绑定在特定模型或会话中

---

## 工作机制

```
文档/代码库（基本载体）
    │
    ├─ 知识点A ←───┐
    │    ├─ 正文   │
    │    └─ 批注（历史记忆）  ←── 记忆附着于知识点
    │             │
    ├─ 知识点B ←───┼─── 知识点之间超链接互联（网状）
    │    └─ 批注   │
    │             │
    └─ [Index/导航索引]
         └─ 通过超链接在不同知识点之间跳转
```

**AI 的工作流**:
1. 读 Index → 找到知识点
2. 读知识点正文 + 该知识点的历史批注（= 记忆）
3. 沿超链接继续深挖下一个知识点
