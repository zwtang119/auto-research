# CDS4WorldCup 球迷视角信息架构重设计

> **日期**: 2026-06-13
> **状态**: 待实现
> **原则**: 首页用对话流回答球迷问题，球队详情页用咨询分析框架构建说服力论证

---

## 第一性原理

球迷进站只有一个问题：**"谁能赢？"** 点进某支球队时问题变成：**"这支队到底能不能夺冠？"**

当前站点以"球队"为主轴组织信息（首页 8 张球队卡、球队列表 48 张卡、详情页 10 个 section），但球迷的心智模型不是"我要了解一支球队"，而是从问题出发、逐层深入。

本设计将站点信息架构从"球队视角"转为"球迷视角"：

- **首页**：用对话流回答球迷的核心问题（热门概率 → 路径热力图 → 焦点比赛 → 选球队）
- **球队详情页**：用投资研究报告的论证结构（论点 → 证据链 → 反证 → 关键变量 → 验证方式）

## 首页设计：对话流

## 球队详情页设计：投资研究报告式论证

### 核心原则

球队详情页不是"数据展示页"，是"关于这支队能不能夺冠的论证"。

遵循咨询分析方法论：
1. **先给结论**（论点），用数据支撑
2. **用证据链构建论证**，不是独立展示数据
3. **主动承认反证**，增强可信度
4. **指出决定性变量**
5. **给出验证方式**

### 页面结构（论证链）

```
论点 → 证据链（1→2→3） → 反证 → 关键变量 → 验证方式 → 市场共识 → 附录
```

#### 1. 论点（我们的判断）

| 元素 | 内容 |
|------|------|
| 头部 | 国旗 + 队名 + 组别 + FIFA 排名 + 夺冠概率（大字） |
| 核心判断 | 1-2 句明确结论（如"大概率进四强，夺冠看签运"） |
| 三张概率卡 | 出线概率（绿）/ 四强概率（橙）/ 夺冠概率（红），各配一句话解释 |

#### 2. 证据链

每条证据服务于论点，不是独立展示：

**证据 1：小组出线分析**
- 小组 3 场对阵 + 概率条
- 每场标注定性判断（"应拿下" / "关键战" / "看情况"）
- 一句话说明小组赛结果如何影响后续签表

**证据 2：路径衰减分析**
- 6 阶段路径条（横版：出线→R32→R16→四强→决赛→冠军）
- 显示每步衰减差值（如"-13pt"）
- 对比其他热门队的衰减（如"西班牙四强→决赛只掉 8pt"）
- 标注关键衰减点（⚠）并用数据说明为什么

**证据 3：看多理由**
- 2-3 条支撑论点的理由
- 每条附一句话解释

#### 3. 反证（什么情况会推翻我们的判断）

- 2-3 条可能推翻论点的场景
- 不用 Bull/Bear 并列——直接陈述"什么情况我们判断会错"
- 红色左边框视觉标记

#### 4. 关键变量（决定性变量）

- 黄色背景高亮
- 一句话指出最影响结果的因素
- 一段话解释为什么这个变量比其他因素更重要

#### 5. 验证方式（怎么验证我们的判断）

| 列 | 内容 |
|----|------|
| 赛前盯这些 | 3 条可观察信号 |
| 赛后对账 | 3 条可验证指标（含模型预期值） |

#### 6. 市场共识

- 不用柱状图，用一句话：`Elo 模型 X% · 市场赔率 ~Y% · AI 投票 ~Z% —— [分歧判断]`
- 分歧判断：分歧小="判断一致"、分歧大="有争议"

#### 7. 附录（折叠区）

点击展开：AI 多视角推演、意外因素分析、完整论述文本、方法论说明

### 从当前页面移除/合并的区块

| 旧区块 | 处理 |
|--------|------|
| Thesis + References | 合并到"论点" |

## 数据结构需求

### 焦点比赛算法

```
对每场比赛：
  gap = max(
    |odds.home_win - coach.home_win|,
    |odds.draw - coach.draw|,
    |odds.away_win - coach.away_win|
  )
按 gap 降序排列，取 Top 6
```

### 路径衰减计算

```
对每支球队：
  stages = [qual_prob, r32_prob, r16_prob, qf_prob, final_prob, champion_prob]
  drops = [stages[i] - stages[i+1] for i in range(5)]
  关键衰减点 = max(drops) 的位置
```

### 球队详情页"市场共识"

从 `baselines-data.js` 提取该队的 Elo/市场/AI 三个概率值，计算 max - min 作为分歧度。

## 文案规范

### 首页文案原则

1. **eyebrow 用 2-4 个字**："谁走得更远""哪场最悬""找到你的队""换个角度"
2. **h2 用完整句子**：包含具体信息，不是空泛标题
3. **描述用陈述句**：不用反问句（"颜色越深越稳" 比 "谁走得更远？" 好）
4. **用具体数字代替形容词**："差了 28 个百分点" 比 "最凶" 有说服力
5. **动词用口语**："看看""卡在哪""走不走得通""对账"
6. **不用翻译腔**：不说"拆解""夺冠路分析"——说"拆开看""能不能夺冠"

### 球队详情页文案原则

1. **论点用陈述句**：如"大概率进四强，夺冠看签运"
2. **证据有因果链**：不是"出线 92%"→ 是"J 组对手实力差距明显，92% 是所有热门队中最高的之一"
3. **反证直接**：不说"看空理由"——说"什么情况会推翻我们的判断"
4. **关键变量一句话**：不超过 15 个字
