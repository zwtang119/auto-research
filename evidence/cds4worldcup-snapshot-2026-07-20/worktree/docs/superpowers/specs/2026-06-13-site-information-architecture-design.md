## 第一性原理：球迷进站后想什么？

一个球迷打开这个网站，心里只有三个问题：

```
Q1: "谁能赢？"          → 核心答案（概率/排名/热门）
Q2: "我支持的队行不行？"  → 深入探索（球队详情/路径分析）
Q3: "你怎么知道的？"      → 方法论/来源（可信度/数据来源）
```

**设计原则**：信息架构必须按 Q1→Q2→Q3 的递进顺序组织。球迷先看到答案，再选择深入，最后才关心方法论。

## 首页信息递进逻辑（关键改动）

### 当前问题
首页 8 个区块的递进混乱：
- Hero（答案）→ 选球队 → AI视角 → 进度 → 难点 → 外界 → 方法 → 更新
- 球迷在 Hero 之后立刻被拖入"AI视角"和"进度"，而不是继续深入核心答案

### 设计递进

```
Layer 1: 核心答案（Q1: 谁能赢？）
  ├── Hero 概率卡 ─── 5 秒内建立关联
  └── 路径热力图 ─── "谁走多远"一屏看完
         ↓ 点击某队

Layer 2: 深入探索（Q2: 我支持的队行不行？）
  ├── 选球队入口 ─── 引导进入具体球队
  ├── AI 多视角 ─── "为什么这么判断"
  ├── 难点分析 ─── "卡在哪"
  └── 外界参照 ─── "别人怎么看"
         ↓ 继续浏览

Layer 3: 方法论（Q3: 你怎么知道的？）
  ├── 方法和来源 ─── 来源分级说明
  ├── 当前进度 ─── 项目状态
  └── 更新记录 ─── 项目动态
```

**首页区块新顺序**：Hero → 热力图 → 选球队 → AI视角 → 难点 → 外界 → 方法来源 → 进度 → 更新记录
